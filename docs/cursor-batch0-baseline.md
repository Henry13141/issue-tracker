# Cursor V3 · Batch 0 实施基线

**日期**：2026-04-10  
**依据**：[cursor-execution-plan-v3.md](./cursor-execution-plan-v3.md)、[product-optimization-plan-v3.md](./product-optimization-plan-v3.md)

## 已确认决策

- 侧栏**保留**一级「我的任务」入口（`/my-tasks`）。
- 成员默认首页维持 `**/home`（工作台）**；管理员登录后根路径 `**/` → `/dashboard`**。

## 页面职责表


| 路由                         | 角色    | 职责                                      |
| -------------------------- | ----- | --------------------------------------- |
| `/`                        | 全部    | 已登录则按角色重定向（admin → dashboard，成员 → home） |
| `/home`                    | 成员为主  | 今日概览统计、建议优先处理列表、与我相关动态、快捷入口             |
| `/my-tasks`                | 全部    | 我负责的未关闭工单分组（今日待同步 / 已推进等）、关注事项          |
| `/issues`                  | 全部    | 团队问题列表、筛选、Tab、新建/导入                     |
| `/issues/[id]`             | 全部    | 单条问题详情、子任务、时间线、附件、事件                    |
| `/reminders`               | 全部    | 协作提醒列表；管理员可看更广提醒                        |
| `/dashboard`               | admin | 管理驾驶舱：统计、风险、成员负载、通知健康                   |
| `/dashboard/notifications` | admin | 外发消息投递记录与重试                             |
| `/members`                 | admin | 成员与企业微信 userid 等                        |
| `/seedance`                | 全部    | Seedance 2.0 独立能力                       |


## 重叠能力

- **「今日待办」**：`/home` 的「今日建议先处理」与 `/my-tasks` 的「今日待同步」均围绕负责人+今日进展，语义重叠；**分工**：home 做概览与入口，my-tasks 做完整列表与操作。
- **快捷入口**：侧栏与 `WorkbenchQuickActions` 均链向 我的任务 / 问题列表 / 待你回应，属有意重复（移动端与侧栏场景）。

## 低风险可直接改动项

- 侧栏「成员与钉钉」→ 企业微信相关文案。
- 侧栏「看板总览」与驾驶舱页 H1「管理驾驶舱」对齐命名。
- `my-tasks` 服务端对「今日是否已更新」由 N 次查询改为批量查询。

## 结构级改动（需单独批次）

- 提醒中心可执行化（列表内写进展）。
- 抽取通用「快速更新」组件并多页复用。
- 驾驶舱「待干预」区块、企业微信健康页、埋点。

## 待产品拍板（未强制）

- 是否在 `/home` 显著增加「去我的任务看完整列表」引导（Batch 2 已加 copy）。
