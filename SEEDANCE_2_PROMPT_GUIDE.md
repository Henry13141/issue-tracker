# Seedance 2.0 提示词指南学习笔记

基于火山方舟官方文档整理：
- 官方文档：<https://www.volcengine.com/docs/82379/2222480?lang=zh>
- 文档标题：`Seedance 2.0 系列提示词指南`
- 最近更新时间：`2026-04-02 18:32:19`

本文是面向项目落地的学习笔记，不是官方原文搬运。重点保留了可直接用于产品接入、提示词编写和工作台设计的要点。

## 1. 文档核心结论

Seedance 2.0 以及 `Seedance 2.0 fast` 原生支持音频与视频联合生成，重点能力有：

- 深度理解自然语言逻辑
- 支持图像、视频、音频多模态参考
- 支持常见文字生成场景
- 支持视频编辑，包括增删改、前后延长、轨道补齐

这份指南的核心思想不是“死记模板”，而是：

- 先明确主体、动作、场景和风格
- 再通过图片/视频/音频参考素材锁定确定性
- 最后用镜头、音效、字幕、特效等进阶描述增强结果质量

## 2. 总体提示词方法

### 2.1 基础公式

官方强调，Seedance 2.0 对自然语言逻辑的遵循度较高，提示词可以按下面的思路组织：

- 谁在做什么
- 在什么环境里
- 画面和风格是什么
- 镜头如何运动
- 是否有音频、氛围声、字幕、特效

可以理解成一条从“主体动作”到“视听细节”的链路：

`主体/对象 + 动作/事件 + 场景/背景 + 风格/光影 + 镜头语言 + 音频/字幕/特效`

### 2.2 多模态参考的指代原则

当上传多张图片、多段视频或多段音频时，官方建议：

- 按使用顺序上传
- 在提示词里明确写 `图片1`、`图片2`、`视频1`、`视频2`
- 明确说明“参考什么”

例如：

- `画面参考图片1的构图`
- `动作参考视频2`
- `角色形象参考图片3`
- `背景音乐氛围参考音频1`

结论：**不是只写“参考这张图”就够了，而是要明确“参考对象 + 被参考属性”**。

## 3. 文字生成能力

官方明确提到，Seedance 2.0 支持在这些场景下生成常用文字：

- `T2V` 文生视频
- `I2V` 图生视频
- `R2V` 参考生视频
- `V2V` 视频生视频

同时支持指定：

- 文字颜色
- 文字风格
- 出现方式
- 出现时机
- 出现位置

官方还特别提醒：

- 优先使用常用字
- 避免生僻字
- 避免特殊符号

### 3.1 广告语 / Slogan

推荐模板：

```text
「文字内容」+「出现时机」+「出现位置」+「出现方式」，「文字特征（颜色、风格）」
```

可控要素：

- 何时出现
- 在哪出现
- 如何出现
- 字体和颜色风格

适合：

- 品牌视频
- 宣传片尾板
- 产品口号展示

### 3.2 字幕

推荐模板：

```text
画面底部出现字幕，字幕内容为“……”，字幕需与音频节奏完全同步。
```

适合：

- 旁白视频
- 对话视频
- 解说类视频

重点是同步关系要说清楚，比如：

- `字幕需与画外音完全同步`
- `画面底部按照台词出现字幕`

### 3.3 气泡台词

推荐模板：

```text
「角色」说：“……”，角色说话时周围出现气泡，气泡里写着台词。
```

适合：

- 漫画风
- 动画风
- 社交媒体短视频

## 4. 图片参考

文档强调，Seedance 2.0 不只是“单图参考”，还支持：

- 主体多视角参考
- 多图元素组合
- 场景图参考
- 分镜图参考
- 多宫格分镜参考

### 4.1 主体多视角图参考

推荐模板：

```text
参考/提取/结合「图片n」中的「主体」，生成「画面描述」，保持「主体」特征一致。
```

适合对象：

- 商品
- 人物
- 角色
- 物件

典型用途：

- 电商产品展示
- 角色一致性生成
- 单主体多角度镜头

要点：

- 如果有多张主体图，最好覆盖不同角度
- 明确要求“保持主体特征一致”
- 如果要强调展示过程，可再补镜头描述

### 4.2 多图参考

推荐模板：

```text
参考/提取/结合/按照/生成「图片n」中的「被参考元素描述」，生成「画面描述」，保持「被参考元素」特征一致。
```

文档中给出的典型子类型有：

- `Logo 参考`
- `多主体参考`
- `多元素参考`
- `多宫格分镜图参考`
- `分镜图参考`

适合理解为：

- 图片1 可以管 Logo
- 图片2 可以管角色
- 图片3 可以管服装
- 图片4 可以管场景
- 图片5 可以管角标或标识

这意味着提示词里最好写成“分工明确”的形式，而不是笼统说“参考这些图片”。

### 4.3 分镜图参考

这是文档里非常实用的一类。

推荐写法：

- 明确哪张图片负责哪一段构图
- 明确镜头切换顺序
- 明确角色是谁参考哪张图

适合：

- 故事短片
- 广告分镜复现
- 对话场景
- 动作场景

如果要生成“按顺序出现”的镜头，提示词里最好加：

- `各个分镜构图按照顺序出现`
- `接着切到图片3的构图`
- `随后镜头切回人物特写`

## 5. 视频参考

官方将视频参考分成三类，分别对应三个不同维度。

### 5.1 动作参考

推荐模板：

```text
参考「视频n」的「动作描述」，生成「画面描述」，保持动作细节一致。
```

适合：

- 角色动作迁移
- 打斗动作复现
- 动物运动姿态参考
- 商品动态展示节奏参考

关键不是只说“参考视频1”，而是：

- 参考哪个动作
- 哪些动作细节要一致

### 5.2 运镜参考

推荐模板：

```text
参考「视频n」的「运镜描述」，生成「画面描述」，保持运镜一致。
```

适合：

- 航拍式俯冲
- 推拉摇移跟等镜头运动
- 第一视角沉浸式镜头
- 宣传片镜头语言迁移

关键点：

- 区分“动作参考”和“运镜参考”
- 如果参考的是镜头，而不是人物动作，就直接写“参考视频1的运镜”

### 5.3 特效参考

推荐模板：

```text
参考「视频n」的「特效描述」，生成「画面描述」，保持特效一致。
```

适合：

- 粒子特效
- 翅膀生成轨迹
- 能量光效
- 爆炸、烟雾、闪光等效果

关键点：

- 指明参考的是“特效”
- 指明特效应用到哪个对象身上
- 如果需要一致轨迹，也要明确写出

## 6. 视频编辑能力

Seedance 2.0 不只是生成，也支持编辑已有视频。

官方列出的编辑能力包括：

- 增加元素
- 删除元素
- 修改元素
- 向前延长
- 向后延长
- 轨道补齐

### 6.1 元素增删改

推荐模板：

```text
增加元素：在「视频n」的「时间位置」+「空间位置」，增加「理想元素描述」。
删除元素：删除「视频n」中的「被删除元素」，视频其他内容保持不变。
修改元素：将「视频n」中的「被更换元素描述」，替换为「理想元素描述」。
```

使用建议：

- 增加元素时最好说清时间和空间位置
- 删除元素时加一句“其他内容保持不变”
- 替换元素时加一句“动作和运镜不变”

这三种约束能显著提高结果稳定性。

### 6.2 视频延长

推荐模板：

```text
- 向前/向后延长「视频n」+「需延长的视频描述」
- 生成「视频n」之前/之后的内容 + 「需延长的视频描述」
```

官方说明：

- 模型会自动截取衔接部分进行合成
- 输入视频原有片段不会重复生成

这很关键，意味着提示词不需要再描述“重复原视频”，而是要描述：

- 前面发生了什么
- 后面发生了什么
- 如何自然承接

### 6.3 轨道补齐

推荐模板：

```text
「视频1」+「过渡画面描述」+接「视频2」+「过渡画面描述」+接「视频3」
```

官方限制：

- 最多支持 `3` 段视频输入
- 总时长不得超过 `15` 秒

官方还说明：

- 生成时会自动截取首尾视频的衔接部分
- 仅保留必要片段参与合成

所以在产品侧要注意：

- 不要让用户无限加视频
- 视频总时长要提前校验
- 对轨道补齐的提示词，应强调“过渡方式”而不是重复素材内容

## 7. 适合产品化的提示词模板

下面是结合官方文档整理后的可落地模板。

### 7.1 通用生成模板

```text
主体是「主体描述」，正在「动作描述」。
场景位于「场景描述」，整体风格为「风格描述」，光线为「光影描述」。
镜头采用「运镜描述」。
如有声音，「音频描述」。
如有字幕/文字，「文字出现方式与位置描述」。
```

### 7.2 图片参考模板

```text
参考图片1中的「主体/构图/服装/Logo/场景」，
生成「目标画面描述」，
保持「被参考元素」特征一致。
```

### 7.3 视频动作参考模板

```text
参考视频1的「动作描述」，
生成「目标画面描述」，
保持动作细节一致。
```

### 7.4 视频运镜参考模板

```text
参考视频1的运镜，
生成「目标画面描述」，
整体镜头运动方式保持一致。
```

### 7.5 视频编辑模板

```text
将视频1中的「原元素描述」替换为「新元素描述」，
其他内容保持不变，动作和运镜不变。
```

## 8. 工作台接入启发

如果把这篇指南继续转成产品能力，建议在工作台里提供这些结构化输入区：

- `主体 / 动作`
- `场景 / 风格`
- `镜头语言`
- `声音 / 字幕`
- `图片参考说明`
- `视频参考说明`
- `编辑模式（增删改 / 延长 / 补齐）`

还可以做一些预设 chips：

- `动作参考`
- `运镜参考`
- `特效参考`
- `字幕同步`
- `气泡台词`
- `保持主体一致`
- `动作和运镜不变`

这样用户就不用从零写长提示词，而是能按文档思路组装。

## 9. 实用写法总结

这篇文档最值得落实的，不是单个案例，而是下面几个写法习惯：

- 写清楚“谁、在做什么”
- 明确参考素材编号
- 明确参考的是“主体 / 构图 / 动作 / 运镜 / 特效 / Logo”
- 多图、多视频时写清顺序
- 做编辑任务时补上“不变项”
- 做字幕/台词任务时强调同步关系
- 避免生僻字和特殊符号

## 10. 一句话总结

Seedance 2.0 的高质量提示词，不是堆砌形容词，而是把：

- 主体
- 动作
- 场景
- 风格
- 参考对象
- 镜头
- 音频
- 编辑约束

按清晰的逻辑关系写出来。

当参考对象和不变约束写得足够明确时，生成结果会稳定很多。
